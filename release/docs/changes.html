Changes in 1.1

Jace will register a shutdown hook jace.util.ShutdownHook to prevent interaction with the JVM after it has shutdown (otherwise we get a core-dump). If you register your own shutdown hooks that use Jace, you should invoke 
Runtime.removeShutdownHook(jace.util.ShutdownHook.getInstance());
on startup and 
jace.util.ShutdownHook.getInstance().run();
at the end of your shutdown hook.
Upgraded to Retroweaver 1.2.2. Java 1.3 users must now include retroweaver-rt-1.2.2.jar in their classpath.
Provided jace.jar for Java5 users and jace-1.3.jar for Java 1.3 users
So in summary, when using Jace to generate proxies or peers you must include jace-1.3.jar and retroweaver-rt-1.2.2.jar in your classpath. And at runtime, your application must include jace-runtime.jar in its classpath.

Changes in 1.1 RC1

Fixed many, many, many bugs, including all those listed in SourceForge, and many not listed in SourceForge.
Added local reference management. Now you can run your programs forever without leaking any memory or resorting to Push/PopLocalFrame, DeleteLocalRef, or DetachCurrentThread calls.
Added a new -mindep option to AutoProxy. With this new option, AutoProxy now only generates the bare minimum set of classes for your code to correctly compile. This reduces the number of classes generated to 1/10th the original on average. For example, map_example now contains 7 auto-generated classes, down from 75.
Added guard checking for faster compilation. Now Jace builds in 20 seconds, and all of the shipped examples build in under 30 seconds.
Added JACE_CHECK_NULLS - You can turn this on to get exceptions when you dereference null Java objects. See the new array example.
Added JACE_CHECK_ARRAYS - You can turn this on to get exceptions when you do silly things with arrays. See the new array example.
Added java_cast and instanceof template functions that act like their Java counterparts. Check out javacast.h and the map_example. These functions type-safely replace JObject.getJavaJniObject(), JObject( jobject ), and JObject( jvalue ).
Added STL conforming random access iterators to JArray. See the new array example.
Fixed support for VC++.NET (now works out of the box with VC++7.0 and VC++7.1)
Fixed support for gcc 3.2 (now works out of the box)
Added support for the Comeau compiler
Added two new examples - array_example and peer_singleton
Added a new BatchEnhancer utility


Changes in 1.1 Beta3

New support for Peers. Nifty byte code enhancement and code generation now takes all of the manual effort out of this task.
New support for dynamic VM loading. Thanks to Philipp Seith for some donated code related to Win32.
New support for specifying options to the virtual machine through the createVm() function.
New support for gcc 3.0+ on MacOSX.
New support for VC++ .NET (7.0).
The Developer's Guide has been updated and also contains several new examples.
Fixed some bugs that were preventing Jace from compiling correctly with Forte C++.
Changed the names of the getJavaObject, getJavaClass, and staticGetJavaClass functions so that they don't clash with other libraries (like Castor's JDO).
Fixed some naming conflicts between proxies in the Java library and common Linux macros.
Removed the dependency on JDK 1.4 from the Jace tools.
Fixed BatchGenerator so that it generates classes in the correct directories.


Changes in 1.1 Beta2

Removed all dependencies from the JDK from the Jace C++ runtime library. You may now easily use Jace with any version of any JDK.
Jace now supports gcc 3.0+ and Linux.
Added the new AutoProxy tool. It scans C++ source and header files and automatically generates all of the necessary C++ Proxies.
Jace now sports a nifty Programmer's Guide.
Increased support for Internationalization
Fixed a bug where ~JClassImpl could throw.
Added a new createVM() utility function.


Changes in 1.0.3a

Migrated the few classes in the framework library to the jnilib library.
Fixed a problem with BatchGenerator where it was omitting jace/ from the path in jace/java* classes.
Fixed a problem with BatchGenerator and ProxyGenerator where they were allowing illegal C++ characters (like '$') in Java class names to propagate unchanged to the C++ proxy classes.
Fixed a bug in ProxyGenerator where generated classes didn't inherit interfaces virtually.
Changed ProxyGenerator so that generated interfaces now have a no argument constructor that delegates to Object::Object( NO_OP ).
Changed ProxyGenerator so that classes no longer specify interfaces in their constructor initializer list.
Fixed a bug in ProxyGenerator where inherited superclasses and interfaces weren't fully-qualified, sometimes resulting in a name clash.
The debug version of jnilib is now named jnilibd to distinguish it from the release version which is named jnilib.


Changes in 1.0.3

Restructured build process.
Separated JDK class library from the Jace core.
Created separate source and binary releases.
Did some serious documenting.


Changes in 1.0.2

Added JObject::getLocalObject(). See the new documentation for JObject::getLocalObject() for more details.
Fixed a bug in which JClassImpl was allowing its jclass references to go out of scope. JClassImpl now creates a global reference to its jclass.


Changes in 1.0.1

Fixed a bug in JField.get() which was trying to construct a JFieldProxy using a jobject instead of a jvalue.
